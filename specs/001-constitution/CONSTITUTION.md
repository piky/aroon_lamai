# Project Constitution: aroon_lamai

**Project Name**: aroon_lamai (Restaurant Order Management System)  
**Created**: December 28, 2025  
**Status**: Active  

## Mission Statement

Build a collaborative restaurant order management system that enables seamless communication between waitstaff and kitchen personnel, while offering customers a modern self-service ordering experience.

## Core Values

### 1. Efficiency First
- Minimize order processing time from customer to kitchen
- Reduce verbal communication errors between waitstaff and chefs
- Enable real-time order tracking and status updates

### 2. Accessibility
- Works on personal smartphones (iOS/Android) for all users
- No app installation required for customer QR code ordering
- Simple, intuitive interface for non-technical users

### 3. Reliability
- Orders are never lost or forgotten
- Real-time notifications ensure timely preparation
- Offline-capable order queuing

### 4. Collaboration
- Waitstaff and kitchen staff work as a unified team
- Clear order status visibility for all stakeholders
- Instant notification of order changes or cancellations

## Design Principles

### User Experience
- **Waitstaff**: Quick order entry, real-time status updates, seamless bill generation
- **Customers**: Intuitive mobile ordering via QR scan, easy customization
- **Kitchen Staff**: Clear order display, priority management, completion notifications

### Technical Guidelines
- Mobile-first responsive design
- Low latency order transmission
- Graceful handling of network interruptions
- Secure order and customer data

## Scope

### In Scope
- Mobile-based order taking for waitstaff
- QR code customer self-service ordering
- Real-time kitchen order notifications
- Order status tracking (pending → preparing → ready → served)
- Bill generation and payment integration

### Out of Scope (Phase 1)
- Inventory management
- Employee scheduling
- Multi-location support
- Advanced analytics/reporting
- Integration with third-party delivery services

## Success Criteria

1. Order placement to kitchen notification: < 3 seconds
2. Customer self-service order completion: < 2 minutes
3. Kitchen staff can acknowledge orders with one tap
4. Bill generation within 30 seconds of request
5. System supports concurrent orders from multiple waitstaff

## Stakeholders

| Role | Primary Needs |
|------|---------------|
| Waitstaff/Waitress | Fast order entry, status updates, bill generation |
| Chef/Cook | Clear order display, priority management, notifications |
| Customers | Easy ordering, customization, order tracking |
| Restaurant Owner | Reliability, efficiency, reporting (future) |

---

**Constitution established** — All project decisions should align with these principles.
